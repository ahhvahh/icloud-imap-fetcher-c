# Contribuindo

## Objetivo

Este projeto prioriza:

- simplicidade operacional;
- compatibilidade com Linux;
- baixo acoplamento;
- legibilidade;
- facilidade de auditoria.

## Regras gerais

- evite dependências desnecessárias;
- prefira manter o projeto em C11;
- documente qualquer alteração relevante;
- não envie credenciais, anexos reais ou logs com dados sensíveis;
- preserve compatibilidade com execução não interativa.

## Fluxo sugerido

1. crie uma branch;
2. implemente a alteração;
3. atualize a documentação afetada;
4. valide o build local com make clean && make;
5. abra um pull request com descrição objetiva.
